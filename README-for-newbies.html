README for newbies

sql-mcp-powershell — open in a browser. Companion to README.md.

Step by step (newbies)

  1. DoClone the repo and open that folder in Cursor (so the project is the workspace).
  2. DoCopy .env.example → .env (same folder as the README). Edit server, database, trust cert, optional SQL user/password.
  3. Needsqlcmd must work in PowerShell (Get-Command sqlcmd). If missing, install Microsoft SQL command-line tools first.
  4. DoReload MCP or restart Cursor. Ask the AI to run SELECT 1 using the SQL tool — you should see a small result, not an error.

Secrets live only in .env (gitignored). Never paste passwords into chat.

Flow (what happens)

Files (color map)

  • Repo.cursor/mcp.json — tells Cursor how to start the server.
  • Codemcp/sql-server-run/sql-server-run.ps1 — reads .env, speaks MCP, calls sqlcmd.
  • Local.env — your machine only; not in git.
  • Copy.env.example — safe template to duplicate.
  • IdeasSUGGESTED-PROMPTS.md — example prompts for the agent.

What to ask the agent

Full copy-paste wording → SUGGESTED-PROMPTS.md (repo root).

  • Quick — List tables/views; describe columns for one table; map PK/FK relationships.
  • Quality — Profile nulls/distincts; find orphan FK rows; scan dates for impossible values.
  • Modeling — Propose Kimball-style star schema views; SCD strategy; grain statements for facts.
  • Big project — Full DB inspection → new mart schema (views) → HTML wireframe with tabs, colors, diagrams, plus executive summary (read-only SQL unless you allow DDL).
  • Ops / reports — Index ideas; large-table summary; permission audit; metric dictionary or cohort SQL.

If something breaks

  • “sqlcmd” not found — install tools, reopen terminal, check PATH.
  • Login / TLS errors — fix SQL_SERVER, user/password, or try SQL_TRUST_CERT=1 for dev only.
  • MCP idle / no tools — workspace must be the repo root; reload MCP after editing mcp.json.
